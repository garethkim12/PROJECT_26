# 구글 맵스 리뷰 크롤러 — 후쿠오카 현 조별과제

구글 맵스에서 장소를 검색해 리뷰(작성자, 별점, 날짜, 본문)를 CSV로 저장하는 스크립트입니다.
**목표: 1인당 10만 건.** 중간 저장과 이어하기를 지원하므로 며칠에 나눠 돌려도 됩니다.

## 담당 주제

| 담당자 | 주제 | 기본 카테고리 |
|---|---|---|
| 김동건 | 관광 | 관광, 쇼핑, 오락, 온천 |
| 김근하 | 숙박 | 호텔, 료칸, 게스트하우스 |
| 남종희 | 음식점 | 맛집, 라멘, 이자카야, 카페 |

## 준비물 (최초 1회)

1. **Python 3.10 이상** + **크롬 브라우저** 설치
2. 패키지 설치:
   ```
   pip install selenium pandas
   ```
   (크롬 드라이버는 Selenium이 자동으로 받아주므로 따로 설치할 필요 없음)

## 사용법

자기 주제 이름만 바꿔서 실행하면 됩니다:

```
python main.py --topic 관광 --max-places 40 --max-reviews 5000
python main.py --topic 숙박 --max-places 40 --max-reviews 5000
python main.py --topic 음식점 --max-places 40 --max-reviews 5000
```

- `--max-places`: 카테고리당 수집할 장소 수 (기본 40)
- `--max-reviews`: 장소당 수집할 리뷰 수 상한 (기본 5000)

실행하면 크롬 창이 자동으로 뜨는데 **닫지 말고 그대로 두세요.**
수집 속도는 분당 약 300건 정도입니다 (리뷰가 많이 쌓인 장소는 점점 느려짐).

### 10만 건 모으는 방법

- 한 번에 다 모을 필요 없습니다. **장소 하나가 끝날 때마다 CSV에 바로 저장**되고,
  진행 상황이 `output/progress_<주제>.json`에 기록되므로 **같은 명령을 다시 실행하면
  이미 끝난 장소는 건너뛰고 이어서** 수집합니다. 중간에 꺼져도 끝난 장소까지는 안전합니다.
- 자기 전에 켜 두는 식으로 2~4일에 나눠 돌리는 것을 권장합니다 (10만 건 ≈ 6~12시간).
- 다 돌렸는데 10만 건이 안 되면:
  1. `--max-places`를 늘려서 재실행 (이미 한 장소는 자동으로 건너뜀)
  2. `TOPICS` 딕셔너리에 카테고리를 추가 (예: 관광이면 "신사", "공원", "박물관" 등)
  3. `--keyword "후쿠오카 현 신사"` 처럼 일회성 검색어로 보충 — 단, 이 경우
     별도 CSV로 저장되므로 마지막에 합치면 됩니다.
- 처음부터 다시 하려면 `--restart` (기존 CSV는 지우지 않으니 직접 삭제).

### 그 외 옵션

```
# 주제 안의 카테고리 하나만
python main.py --topic 관광 --category 온천

# 프리셋에 없는 검색어로 직접
python main.py --keyword "후쿠오카 현 야타이"
```

### 검색 키워드 바꾸기

스크립트 상단의 `TOPICS` 딕셔너리를 수정하면 됩니다.
예: 숙박에 "캡슐호텔"을 추가하려면 `"숙박"` 항목에
`"캡슐호텔": f"{REGION} 캡슐호텔",` 한 줄을 추가.

## 출력

`output/reviews_<주제>.csv` 에 누적 저장됩니다 (재실행해도 같은 파일에 이어붙음).
utf-8-sig 인코딩이라 엑셀에서 바로 열립니다.

| 컬럼 | 내용 |
|---|---|
| category | 검색 카테고리 (온천, 라멘 등) |
| place_name | 장소명 |
| reviewer | 작성자 |
| rating | 작성자가 남긴 별점 (1~5) — 감성분석 레이블로 사용 가능 |
| date | 작성 시기 ("3개월 전" 형식) |
| text | 리뷰 본문 |

## 자주 생기는 문제

- **"리뷰 탭을 찾지 못했습니다" 경고** — 구글이 가끔 다른 화면을 보여줘서 생기는데,
  자동으로 3회 재시도합니다. 계속 실패한 장소는 건너뜁니다.
- **장소당 리뷰가 5000건에서 멈춤** — `--max-reviews 10000` 처럼 상한을 올리면 됩니다.
  단, 한 장소에서 리뷰를 많이 모을수록 스크롤이 점점 느려지므로
  장소 수를 늘리는 쪽이 보통 더 빠릅니다.
- **차단 걱정** — 사람이 보는 화면을 그대로 스크롤하는 방식이라 과제 수준에서는
  보통 문제없지만, 한 PC에서 여러 개를 동시에 띄우지는 마세요.
